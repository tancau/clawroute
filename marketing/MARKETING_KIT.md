## ❓ 常见问题解答

### Q1: HopLLM 会影响响应质量吗？
**A:** 不会。HopLLM 的智能路由会根据任务复杂度选择合适的模型。简单任务用轻量模型足够，复杂任务仍然用高级模型，确保质量不变。

### Q2: 和直接用 OpenRouter 有什么区别？
**A:** OpenRouter 是模型聚合平台，需要手动选择模型。HopLLM 自动帮你选择最优模型，省去决策成本。

### Q3: 数据安全吗？
**A:** HopLLM 开源可自部署，数据完全在你的控制下。托管版本也不存储对话内容，只做路由转发。

### Q4: 支持哪些模型？
**A:** 支持 Qwen、DeepSeek、Claude、GPT-4、Gemini、Llama 等 24+ 主流模型，持续增加中。

### Q5: 如何收费？
**A:** HopLLM 本身免费开源。使用时你只需要支付各模型服务商的费用，按实际 token 使用量计费。

### Q6: 延迟会增加多少？
**A:** 路由判断通常在 10-50ms，相比模型响应时间几乎可忽略。本地部署可进一步降低延迟。

### Q7: 可以自己部署吗？
**A:** 可以。`git clone` 后按 README 部署即可，支持 Docker 部署。

### Q8: 适合什么场景？
**A:** 特别适合：
- 调用量大的应用（客服、交易 Bot）
- 对成本敏感的初创公司
- 需要多模型协作的场景
- 想要优化 API 成本的团队

### Q9: 不懂代码能用吗？
**A:** 在线版本提供可视化配置界面，选择场景后一键导出配置，由技术人员集成即可。

### Q10: 和 LiteLLM 对比如何？
**A:**
| 功能 | HopLLM | LiteLLM |
|------|--------|---------|
| 智能路由 | ✅ 自动 | ❌ 需配置 |
| 意图分类 | ✅ 内置 | ❌ 无 |
| 可视化配置 | ✅ 有 | ❌ 无 |
| 部署难度 | 低 | 中 |
| 开源 | ✅ | ✅ |
